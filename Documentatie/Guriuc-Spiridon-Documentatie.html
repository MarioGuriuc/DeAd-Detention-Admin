Detention Admin Documentation

Mario Guriuc, Vlad Spiridon

Introduction

Purpose

The primary objective of this document is to delineate the comprehensive requirements and specifications for the design and development of a sophisticated web-based application tailored for the efficient management of visitation processes within correctional facilities, encompassing both prisons and correctional homes. This advanced application aims to offer a robust suite of features to meticulously record, manage, and analyze visitor interactions, ensuring transparency, security, and adherence to established protocols.

Scope

Specifically, the application will be designed to:

  • Accurately capture and store detailed information about visitors, including their personal details, contact information, photographic identification, and relationship status with the incarcerated individual.
  • Systematically record comprehensive visitation data, encompassing essential details such as visitation dates, duration, nature of the visit, items exchanged during the visit, summaries of discussions, and evaluations of the inmate's health and mood during the interaction.
  • Facilitate seamless integration of visitor and inmate records, enabling administrators to efficiently manage and associate visitors with specific inmates, ensuring proper tracking and monitoring of all visitation activities.
  • Empower authorized personnel with robust reporting tools to generate in-depth statistical reports, offering valuable insights into visitation trends, patterns, and behaviors over specified time periods, thereby facilitating informed decision-making and policy formulation.
  • Implement a secure, user-friendly interface accessible via standard web browsers on various computing devices, ensuring widespread accessibility while maintaining stringent security measures to safeguard sensitive data and protect user privacy.

Overall Description

Product Features

  • Detention centers Management: Recording information about detention centers, including location and description.With multiple features for administrators, like adding an inmate, deleting the center, or editing existing information.
  • Inmates Management: Recording information about inmates, including name ,facility location, crime committed and the current sentences.Administrator can add, delete or edit the information.They can also transfer an inmate to another facility.
  • Visitor Management: Recording visitor details like identity.
  • Visit Information: Capturing data such as visit date, duration, nature of visit, items exchanged, discussions summary, inmate's health and mood, witnesses, etc.The administrator can also edit the visit information, and the visit status.
  • Statistical Reporting: Generating reports in HTML, CSV, and JSON formats based on various criteria such as individual visits, time periods, etc.
  • API Access: Providing essential functionalities through a REST/GraphQL API for seamless integration with other systems or external applications.

System Features

Docker Containerization

Description

Containerization is a lightweight alternative to full machine virtualization that involves encapsulating an application in a container with its own operating environment. Docker is a popular containerization platform that allows developers to build, ship, and run applications within containers.

The detention administration system will be containerized using Docker to ensure consistency, portability, and scalability across different environments. This will enable the system to be easily deployed and managed in various settings, including development, testing, and production environments.

The Docker containers will encapsulate the application code, dependencies, and configuration settings, providing a self-contained environment for the system to run without interference from external factors. This approach enhances system reliability, security, and performance while simplifying the deployment process for administrators and developers.

Docker Compose will be used to define and manage multi-container Docker applications, allowing the detention administration system to be composed of multiple services, such as the web server, database, and API server. This will streamline the deployment and orchestration of the system components, ensuring seamless integration and operation of the application.

Backend and Database

The backend of the application is developed using PHP, a server-side scripting language known for its versatility and ease of integration. PHP will handle the logic of the application, including data processing, authentication, and communication with the database.

The architecture of the backend is based on microservices, each service being responsible for a specific aspect of the application, such as user management, visitation records, or statistical reporting. This modular approach allows for greater flexibility, maintainability, and scalability, as each service can be developed, tested, and deployed independently.

For the database management system, MongoDB will be utilized. MongoDB is a NoSQL database that offers flexibility and scalability, making it suitable for storing and managing the diverse data structures associated with visitation records. It provides features such as document-based storage, dynamic schemas, and high availability, which are advantageous for the requirements of the detention administration website.

Admin Control

Description

The Admin Control feature provides system administrators with tools to manage and oversee various aspects of the detention center visitation system. This includes approving or denying visitor requests, adding new detention centers and inmates, and generating reports for analysis.

Features

  • Visitor Approval: Allows administrators to review and approve or deny visitor requests based on eligibility and compliance with visitation policies.
  • Add Detention Centers: Enables administrators to add new detention centers to the system, specifying details.The detention centers can be edited or deleted.
  • Add Inmates: Provides functionality for administrators to add new inmates to the system, capturing essential details like name, identification, facility location, and crime committed.The inmates can be edited or deleted.
  • Generate Reports: Allows administrators to generate various reports for analysis, including visitor statistics, inmate details, and visitation trends.

Inputs

  • Visitor requests: Details of visitors requesting access, including name, contact information, and purpose of visit.
  • Detention center details: Information about new detention centers, such as location, capacity, and current occupancy status.
  • Inmate details: Information about new inmates, including name, identification, facility location, and crime committed.
  • Report criteria: Parameters for generating reports, such as date range, type of report, and specific data points to include.

Outputs

  • Visitor approval status: Confirmation of approved or denied visitor requests.
  • New detention centers: Record of newly added detention centers in the system.
  • New inmate records: Confirmation of successfully added new inmates to the system.
  • Generated reports: Various reports in desired formats (HTML, CSV, JSON) providing insights and analysis of system data.

Detention Admin Backend Routes

GET /api/centers

Route for retrieving a list of all detention centers registered within the system. Returns a JSON array of detention center objects containing details such as name, location, and description.

GET /api/center/{center_id}

Route for retrieving a specific detention center based on the center ID. Returns a JSON object containing detailed information about the specified detention center, including name, location and description.

GET /api/centers/count

Route for retrieving the total count of detention centers registered within the system. Returns an integer value representing the total number of detention centers.

GET /api/centers/page/{page_number}

Route for retrieving a paginated list of detention centers registered within the system. Accepts a page number as input to fetch the corresponding subset of detention centers. Returns a JSON array of detention center objects for the specified page.

GET /api/account/{username}

Route for retrieving user account details based on the username. Returns a JSON object containing user information such as name, email, and role.

POST /api/login

Route for user login. Requires username and password as input. Returns a JSON Web Token (JWT) for authentication.

POST /api/register

Route for user registration. Requires user details such as name, email, and password as input. Returns a success message upon successful registration.

POST /api/forgot-password

Route for resetting user password. Requires user email as input. Sends a new randomly generated password to the user's email.

POST /api/{username}/change-password

Route for changing user password. Requires user's current password and new password as input. Returns a success message upon successful password change.

GET /api/verify-jwt

Route for verifying JSON Web Token (JWT) for user authentication. Requires JWT as input. Returns a success message if the JWT is valid, also a refreshed JWT.

GET /api/get-username

Route for retrieving the username. Returns a JSON object with the username information.

GET /api/logout

Route for user logout. Performs logout actions and returns a success message upon completion.

GET /api/centers/{center_id}/inmates

Route for retrieving a list of inmates in a specific detention center based on the center ID. Returns a JSON array of inmate objects with details such as name, crimes, and sentences.

GET /api/centers/{center_id}/inmates/count

Route for retrieving the count of inmates in a specific detention center based on the center ID. Returns an integer value representing the total number of inmates.

GET /api/account/{username}/visits

Route for retrieving visits associated with a specific user account based on the username. Returns a JSON array of visit objects with details such as date, time, and status.

GET /api/statistics/{center_id}/{inmate_id}/{start_date}/{end_date}

Route for retrieving statistics based on specific parameters: center ID, inmate ID, start date, and end date. Returns a JSON object with statistical data.

PATCH /api/account/{username}

Route for updating user account details based on the username. Requires updated user information as input. Returns a success message upon successful update.

PATCH /api/{username}/change-role

Route for changing user role based on the username. Requires new role information as input. Returns a success message upon successful role change.

PATCH /api/visits/{visit_id}/status

Route for updating the status of a visit based on the visit ID. Requires new status information as input. Returns a success message upon successful status update.

PATCH /api/visits/{visit_id}

Route for editing a visit based on the visit ID. Requires updated visit details as input. Returns a success message upon successful edit.

PATCH /api/centers/{center_id}/inmates/{inmate_id}/transfer

Route for transferring an inmate from one detention center to another based on the center ID and inmate ID. Returns a success message upon successful transfer.

PATCH /api/centers/{center_id}/inmates/{inmate_id}/edit

Route for editing inmate details in a specific detention center based on the center ID and inmate ID. Requires updated inmate information as input. Returns a success message upon successful edit.

PATCH /api/centers/{center_id}/edit

Route for editing detention center details based on the center ID. Requires updated center information as input. Returns a success message upon successful edit.

DELETE /api/account/{username}

Route for deleting user account based on the username. Performs account deletion and returns a success message upon completion.

DELETE /api/centers/{center_id}/inmates/{inmate_id}/delete

Route for deleting an inmate from a specific detention center based on the center ID and inmate ID. Performs deletion and returns a success message upon completion.

DELETE /api/centers/{center_id}/delete

Route for deleting a detention center based on the center ID. Performs deletion and returns a success message upon completion.

PUT /api/centers

Route for adding a new detention center. Requires new center details as input. Returns a success message upon successful addition.

PUT /api/centers/{center_id}/add-inmate

Route for adding a new inmate to a specific detention center based on the center ID. Requires new inmate details as input. Returns a success message upon successful addition.

PUT /api/centers/{center_id}/inmates/{inmate_id}/add-visit

Route for adding a new visit for an inmate in a specific detention center based on the center ID and inmate ID. Requires new visit details as input. Returns a success message upon successful addition.
