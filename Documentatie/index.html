Detention Admin Website

Mario Guriuc, Vlad Spiridon

Introduction

Purpose

The primary objective of this document is to delineate the comprehensive requirements and specifications for the design and development of a sophisticated web-based application tailored for the efficient management of visitation processes within correctional facilities, encompassing both prisons and correctional homes. This advanced application aims to offer a robust suite of features to meticulously record, manage, and analyze visitor interactions, ensuring transparency, security, and adherence to established protocols.

Scope

Specifically, the application will be designed to:

  • Accurately capture and store detailed information about visitors, including their personal details, contact information, photographic identification, and relationship status with the incarcerated individual.
  • Systematically record comprehensive visitation data, encompassing essential details such as visitation dates, duration, nature of the visit, items exchanged during the visit, summaries of discussions, and evaluations of the inmate's health and mood during the interaction.
  • Facilitate seamless integration of visitor and inmate records, enabling administrators to efficiently manage and associate visitors with specific inmates, ensuring proper tracking and monitoring of all visitation activities.
  • Empower authorized personnel with robust reporting tools to generate in-depth statistical reports, offering valuable insights into visitation trends, patterns, and behaviors over specified time periods, thereby facilitating informed decision-making and policy formulation.
  • Implement a secure, user-friendly interface accessible via standard web browsers on various computing devices, ensuring widespread accessibility while maintaining stringent security measures to safeguard sensitive data and protect user privacy.

Overall Description

Product Features

  • Visitor Management: Recording visitor details including identity, photograph, relationship with the inmate.
  • Visit Information: Capturing data such as visit date, duration, nature of visit, items exchanged, discussions summary, inmate's health and mood, witnesses, etc.
  • Statistical Reporting: Generating reports in HTML, CSV, and JSON formats based on various criteria such as individual visits, time periods, types of offenses, etc.
  • API Access: Providing essential functionalities through a REST/GraphQL API for seamless integration with other systems or external applications.

System Features

Visitor Management

Description

This feature allows users to manage visitor information, including identity, photograph, and relationship with the inmate.

Inputs

  • Visitor details: Name, contact information, photograph, relationship with the inmate.
  • Inmate details: Identification number, name, facility location, crime committed.

Outputs

  • Visitor records stored in the database for future reference.
  • Ability to associate visitors with specific inmates.

Admin Control

Description

The Admin Control feature provides system administrators with tools to manage and oversee various aspects of the detention center visitation system. This includes approving or denying visitor requests, adding new detention centers and inmates, and generating reports for analysis.

Features

  • Visitor Approval: Allows administrators to review and approve or deny visitor requests based on eligibility and compliance with visitation policies.
  • Add Detention Centers: Enables administrators to add new detention centers to the system, specifying details such as location, capacity, and current occupancy status.
  • Add Inmates: Provides functionality for administrators to add new inmates to the system, capturing essential details like name, identification, facility location, and crime committed.
  • Generate Reports: Allows administrators to generate various reports for analysis, including visitor statistics, inmate details, and visitation trends.

Inputs

  • Visitor requests: Details of visitors requesting access, including name, contact information, relationship with the inmate, and purpose of visit.
  • Detention center details: Information about new detention centers, such as location, capacity, and current occupancy status.
  • Inmate details: Information about new inmates, including name, identification, facility location, and crime committed.
  • Report criteria: Parameters for generating reports, such as date range, type of report, and specific data points to include.

Outputs

  • Visitor approval status: Confirmation of approved or denied visitor requests.
  • New detention centers: Record of newly added detention centers in the system.
  • New inmate records: Confirmation of successfully added new inmates to the system.
  • Generated reports: Various reports in desired formats (HTML, CSV, JSON) providing insights and analysis of system data.

Visit Information

Description

This feature enables users to record detailed information about each visit, including date, duration, nature of visit, items exchanged, discussions summary, inmate's health and mood, witnesses, etc.

Inputs

  • Visit date and time.
  • Duration of visit.
  • Nature of visit (e.g., legal consultation, family visit).
  • Items exchanged between visitor and inmate.
  • Summary of discussions.
  • Inmate's health and mood assessment.
  • Witnesses present during the visit.

Outputs

  • Visitation records stored in the database.
  • Ability to generate reports based on visitation data.

Statistical Reporting

Description

This feature allows users to generate statistical reports analyzing visitation trends and patterns over time.

Inputs

  • Criteria for report generation (e.g., individual visits, time periods, types of offenses).
  • Output format (HTML, CSV, JSON).

Outputs

  • Statistical reports presented in the specified format.
  • Insights into visitation patterns for decision-making purposes.

API Access

Description

This feature provides essential functionalities through a REST API written in PHP, running on an Apache2 server for seamless integration with other systems or external applications.

Inputs

  • API requests for data retrieval, insertion, or modification.
  • Authentication credentials for authorized access. (JSON Web Token)

Outputs

  • JSON's with the requested data or confirmation of successful operations.

Backend and Database

The backend of the application is developed using PHP, a server-side scripting language known for its versatility and ease of integration. PHP will handle the logic of the application, including data processing, authentication, and communication with the database.

For the database management system, MongoDB will be utilized. MongoDB is a NoSQL database that offers flexibility and scalability, making it suitable for storing and managing the diverse data structures associated with visitation records. It provides features such as document-based storage, dynamic schemas, and high availability, which are advantageous for the requirements of the detention administration website.

Detention Admin Frontend Pages

AddCenterPage

The AddCenterPage is a dedicated interface designed to facilitate the addition, modification, and management of detention centers within the system. This page enables administrators to record and maintain comprehensive details about each detention center, including location, capacity, current occupancy status, and associated operational information. It provides a centralized platform for administrators to efficiently manage detention center data, ensuring accuracy, consistency, and compliance with established regulations and standards.

AddInmatesPage

The AddInmatesPage is specifically designed to streamline the process of adding, updating, and managing inmate records within the system. This page allows administrators to capture and store essential details about each inmate, such as name, identification number, facility location, crime committed, and current status within the facility. It offers a user-friendly interface with intuitive data entry fields and validation checks to ensure the accuracy and integrity of inmate records, facilitating effective monitoring, tracking, and management of inmate information.

AdminPage

The AdminPage serves as the central administrative dashboard for system administrators, providing a comprehensive overview of system activities, performance metrics, and user interactions. This page offers a range of administrative controls and functionalities, including managing user access and permissions, monitoring system health and performance, generating reports, and configuring system settings. It serves as a hub for administrators to efficiently oversee and manage all aspects of the web application, ensuring optimal functionality, security, and user satisfaction.

AllVisitsPage

The AllVisitsPage provides a comprehensive list and detailed overview of all recorded visits within the system, facilitating easy retrieval, viewing, and analysis of visitation data. This page allows administrators to filter and search visit records based on various criteria, such as visitor details, inmate information, visitation dates, and visitation rights. It offers interactive features and tools to assist administrators in analyzing visitation trends, patterns, and behaviors, enabling informed decision-making, policy formulation, and operational planning.

ConfigureVisitPage

The ConfigureVisitPage offers advanced functionality for configuring and customizing various visit parameters and settings within the system. This page enables administrators to define and manage visitation rules, schedules, visitation rights, and access levels for visitors and inmates. It provides a flexible and customizable platform for administrators to adapt and modify visitation parameters according to evolving requirements, regulations, and policies, ensuring compliance, consistency, and optimal management of visitation processes.

Centers Page

The Centers Page presents a comprehensive list and detailed profiles of all detention centers registered within the system. This page provides valuable information about each detention center, including the name, location, and a description. After selecting a center, the user is redirected to the inmates page. It offers the ability for administrators to add detention centers.

Home Page

The Home Page serves as the primary landing page for the web application, offering a short description about the application. It serves as a user-friendly gateway to the web application, providing visitors with the information they need to navigate the site effectively and access the various features and functionalities available within the system .

InmatesPage

The InmatesPage displays a comprehensive list and detailed profiles of all inmates registered within the system. This page showcases essential details about each inmate, including name, identification number, facility location, crime committed, current status within the facility, and visitation rights. It offers interactive features and tools to assist administrators in managing and monitoring inmate information, facilitating effective tracking, reporting, and communication between different facilities and administrative personnel.

Login Page

The Login Page provides a user-friendly interface for users to access the web application, requiring authentication credentials (username and password) for authorized access. This page incorporates robust security measures, encryption protocols, and validation checks to safeguard sensitive data, protect user privacy, and prevent unauthorized access.

Register Page

The Register Page allows new users to create accounts for accessing the web application, capturing essential information and setting up user profiles. This page offers a user-friendly registration process, guiding users through the data entry fields, validation checks, and account creation steps. It incorporates robust security measures, encryption protocols, and privacy controls to safeguard sensitive information, protect user privacy, and ensure compliance with data protection regulations and policies.

Account Page

The Account Page serves as the personalized dashboard for registered users, providing them with their account information and settings. This page offers a user-friendly interface with intuitive data entry fields. Additionally, the Account Page includes a logout functionality, enabling users to log out of the web application, terminating their active session. Users can also choose to edit their account information, change their password, delete their account, and if the user is an admin, they can redirect themselves to the Admin Page.

GenerateStatsPage

The GenerateStatsPage is a powerful feature of the web application designed to generate comprehensive statistics and reports on the recorded visits. This page provides users with the capability to generate detailed statistical documents in various formats, including HTML, CSV, and JSON. Users can customize the generated reports based on specific criteria such as visits per individual, time period, type of offense committed, or other relevant categories. The GenerateStatsPage offers an intuitive interface with options to select and filter the desired data, facilitating insightful analysis and informed decision-making for administrators and authorized personnel.

Edit Account Page

The Edit Account Page allows registered users to modify and update their account information and settings within the web application. This page offers an interface with data entry fields for users to edit their personal details and contact information. Users can update their account settings. The Edit Account Page incorporates validation checks and security measures to ensure the accuracy and integrity of user data, safeguarding sensitive information and protecting user privacy.

Change Password Page

The Change Password Page provides users with an interface to change their account passwords within the web application. This page incorporates robust security measures, encryption protocols, and validation checks to safeguard sensitive data and protect user privacy. After updating their password, users receive a popup message, ensuring the successful completion of the password change process. After that, the user is logged out and redirected to the login page.

Detention Admin Backend Routes

GET /api/centers

Route for retrieving a list of all detention centers registered within the system. Returns a JSON array of detention center objects containing details such as name, location, and description.

GET /api/centers/count

Route for retrieving the total count of detention centers registered within the system. Returns an integer value representing the total number of detention centers.

GET /api/centers/{page_number}

Route for retrieving a paginated list of detention centers registered within the system. Accepts a page number as input to fetch the corresponding subset of detention centers. Returns a JSON array of detention center objects for the specified page.

GET /api/account/{username}

Route for retrieving user account details based on the username. Returns a JSON object containing user information such as name, email, and role.

POST /api/login

Route for user login. Requires username and password as input. Returns a JSON Web Token (JWT) for authentication.

POST /api/register

Route for user registration. Requires user details such as name, email, and password as input. Returns a success message upon successful registration.

POST /api/forgot-password

Route for resetting user password. Requires user email as input. Sends a new randomly generated password to the user's email.

POST /api/{username}/change-password

Route for changing user password. Requires user's current password and new password as input. Returns a success message upon successful password change.
